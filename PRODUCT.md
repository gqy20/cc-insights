# Product

## Register

product

## Users

cc-insights 面向高频使用 Claude Code 的开发者、团队维护者和 AI 代理。使用者通常在本机或局域网内查看历史会话、失败来源、Token 消耗和工具调用细节，目标是快速判断 Claude Code 为什么变慢、变贵或频繁失败。

## Product Purpose

cc-insights 把 Claude Code 的历史数据转成可解释的证据、诊断结论和下钻入口。成功的界面应该帮助用户回答：问题集中在哪些项目、命令、工具、模型和 session；下一步应该优化 `CLAUDE.md`、hooks、MCP 配置，还是封装新的工具。

## Brand Personality

克制、可靠、证据优先。界面应像工作台，不像营销页；优先保证扫描效率、筛选联动和异常定位。

## Anti-references

避免大段装饰性说明、营销式 hero、花哨动画、过度卡片化和只展示数量但不给证据的图表。不要把每个分析点拆成新的孤立入口。

## Design Principles

- 一个全局筛选状态驱动整屏分析。
- 先给判断，再给证据和下钻。
- 图表必须回答具体诊断问题。
- 保持命令和界面入口收敛，细节进入结构化下钻。
- 局域网访问默认可用，加载失败要给出可执行原因。

## Accessibility & Inclusion

遵循产品工具的可读性基线：正文对比度不低于 WCAG AA，控件支持键盘焦点，动画尊重 reduced motion。大屏、桌面和窄屏都应保持信息不重叠。
